# nobiscuit - Game Design Document

## Game Overview

nobiscuit は TUI（ターミナル）で動くレイキャスティング 3D 迷路ゲーム。
DOOM / Wolfenstein 風の一人称視点を、ハーフブロック文字と 24bit True Color で完全にターミナル内に描画する。

## Concept

**nobiscuit** = no biscuit = のび助（nobisuke）+ ビスケット

### Story

ドラえもんの「ホーム迷路」のような、無限に続く家の中を彷徨う。
各部屋に散らばるビスケットを全て食べて（消して）この世から無くしていく。
全てのビスケットを消し、迷路を脱出した時、自分がのび助だったことに気づく。タイトル回収がダジャレ。

### Why TUI?

- テクスチャの知識がない → テクスチャ不要の方向に振り切る
- TUI のレイキャスティング 3D は他にほぼ存在しない → それ自体が差別化
- Claude Code はテクスチャ付き 3D のデザインが苦手だった（前回の教訓）
- WebGL/Phaser はホットリロードが効かず開発体験が悪かった

## Game System

### Movement

- Wizardry 風のグリッド移動。1 マスずつ、90 度ずつ
- 位置はタイル中央に固定。壁に引っかかることがない
- 前進・後退・左右回転。後退は向きを変えずに後ろに下がる
- アニメーション付きスライド（瞬間移動ではない）

### Controls

| Key | Action |
|---|---|
| W / Up | 前進（1マス） |
| S / Down | 後退（1マス） |
| A / Left | 左を向く（90度） |
| D / Right | 右を向く（90度） |
| M | ミニマップ表示（3秒間、空腹コスト付き） |
| Enter / Space | ガラガラ確定（開始画面のみ） |
| Y | リトライ（結果画面のみ） |
| N | 終了（結果画面のみ） |
| Q / Esc | 終了 |

### Hunger System

- 空腹ゲージ（1.0 = 満腹、0.0 = 餓死）
- 時間経過で減少（約 50 秒で餓死）
- ビスケットを拾うと 0.15 回復（上限 1.0）
- ゲージは HUD 左上にバーで表示。緑→黄→赤で色が変わる
- スコア・経験値・レベルなし。食うか死ぬか出るか

### Sprites

- **ビスケット**: 丸いクッキー型の AA パターン。壁の 1/4 の高さ。床に接地
- **ゴール**: 同サイズの光る球体。少し浮いている
- 距離に応じて自動スケーリング（近い = 大きい AA、遠い = ドット）
- パターンの透明部分からは背景が見える

### Minimap

- 半透明オーバーレイ（alpha 0.4）。3D 画面が透けて見える
- 画面右端にフラッシュ、上下は中央揃え
- プレイヤーは赤い点、向いている方向は黄色い線
- **普段は非表示**。以下の2つの場面でのみ表示される:
  1. **M キー**: 3 秒間だけ表示。空腹度 0.03 を消費
  2. **ビスケット取得**: 2 秒間の全開示フラッシュ（fog of war 無視）
- **Fog of War**: 歩いた場所（周囲±1マス）だけがミニマップに反映される。未踏は黒
- **デバッグモード**: 環境変数 `NOBISCUIT_DEBUG=1` で従来通りの常時トグル（fog of war なし）

### Galagala (ガラガラ) Opening

ゲーム開始時にガラガラ（福引き器）を回す演出。回転回数で迷路の複雑さが決まる。

- 任意のキーでスピン + カメラシェイク。Enter/Space で確定
- 回数に応じて数字の色が変化（灰→白→黄→赤）
- 回せば回すほど地獄。上限なし
- リトライ時もガラガラからやり直し

| 回数 | 迷路サイズ | 階数 |
|---|---|---|
| 1-2 | 15x13 | 1 |
| 3-4 | 25x19 | 2 |
| 5-7 | 31x25 | 3 |
| 8-10 | 51x41 | 5 |
| 11-15 | 81x61 | 8 |
| 16+ | 121x91 | 12 |

### Win/Lose Conditions

- 全ビスケットを消してゴールに到達 → 脱出（エンディング）
- 空腹ゲージが 0 → 行動不能（ゲームオーバー）

### 演出

**ゲームオーバー（行動不能）:**
1. 画面が3秒かけてフェードアウト（暗転）+ カメラが下にずれる（倒れ込み）
2. 黒画面に "You can no longer move..." を表示
3. 3秒後にリトライ選択（Y/N）

**クリア（脱出）:**
1. 画面が3秒かけてフェードアウト
2. タイトル回収演出（段階表示）: "no biscuit..." → "...nobiscuit..." → "...nobisuke."
3. スコア表示（ビスケット数、経過時間、探索階数）
4. リトライ選択（Y/N）

**共通:**
- フェード中はキー入力を無視（最低3秒の猶予）
- Y でリトライ（迷路を再生成して最初から）、N/Q/Esc で終了

## Windows & Shoji

窓と障子は壁のテクスチャバリエーション。壁と同じく solid で通行不可。壁面の上下に壁テクスチャの枠を残し、中央部分だけ窓/障子テクスチャが描画される（壁に埋め込まれた開口部）。

### 窓（TILE_WINDOW）
- 木枠 + ガラス面 + 十字格子。壁面の上下 15% は壁テクスチャ
- ミニマップ上では青色で表示

### 障子（TILE_SHOJI）
- 木の桟（縦3本×横4本）+ 白い和紙。壁面の上部 20% と下部 30% は壁テクスチャ
- 和紙部分に微かな明暗（向こう側の光が透けている感じ）
- ミニマップ上では白色で表示

### 配置
- 内壁の約 15% が窓候補。そのうち約 70% が窓、30% が障子
- 将来的に「移動可能な壁/窓」として、迷路が動的に変化する仕組みの基盤

## Multiple Floors

- 階数はガラガラの回転回数で決定（1〜12階）。階段で上下移動
- 各階は独立した迷路で、それぞれにビスケットが配置される
- 階段上り（↑矢印スプライト）と階段下り（↓矢印スプライト）
- フロア表示（例: "2F" + ドットインジケータ）はミニマップと同時にのみ表示。普段は秘密
- 階移動時にメッセージ表示（"Going up..." / "Going down..."）。階数は伏せる
- ゴールは最上階のみ（全階を探索する必然性を生む）

## Doors

**廊下ハブ構造**: 野比家のルールとして、全てのドアは廊下に面する。部屋同士が直接つながることはなく、必ず廊下を経由する。階段も廊下にのみ配置。

### ドアの種類

| 種類 | タイル | テクスチャ | 配置条件 |
|---|---|---|---|
| ふすま | TILE_DOOR_FUSUMA | 白和紙+引手金具 | 大きい部屋 (面積 ≥ 9: 3x3, 4x3, 4x4, 5x3, 5x4) |
| 台所ドア | TILE_DOOR_KITCHEN | 木目+ドアノブ | 中くらいの部屋 (3x2, 2x3) |
| トイレドア | TILE_DOOR_TOILET | 暗い木目+すりガラス小窓 | 小さい部屋 (2x2) |
| 玄関ドア | TILE_DOOR_GENKAN | 重厚な暗い木+パネル溝 | ゴール最寄りの部屋に1つ |

### 開閉メカニクス

- **閉じた状態**: solid（通行不可）、壁として描画
- **自動オープン**: プレイヤーの隣接4セルにドアがあれば自動で開く
- **自動クローズ**: プレイヤーからマンハッタン距離3以上離れると自動で閉じる
- **表裏一貫性**: どの方向から見ても同じドアテクスチャ（hit side による差は陰影のみ）

## Minimap Visibility

ミニマップの表示は制限されており、「迷子の恐怖感」を演出する。

### 通常モード
- ミニマップは普段は非表示
- **M キー**: 3 秒間のタイマー表示。空腹度 0.03 を消費。情報を得るには代償がある
- **ビスケット取得**: 2 秒間の全開示フラッシュ。fog of war を無視して全タイルを表示。ビスケットの価値が「食料 + 情報」の二重になる
- **Fog of War**: プレイヤーが歩いた場所（周囲±1マス）だけがミニマップに反映される。未踏の領域は黒

### デバッグモード
- 環境変数 `NOBISCUIT_DEBUG=1` で有効
- M キーで従来通りの常時トグル（コストなし、タイマーなし、fog of war なし）

## Maze Generation

- **不定形マスクベース**: 正方形固定ではなく、各階で異なる有機的な形状
- マスク生成: 2-4 個のシード点から BFS でアメーバ状に拡張（全ノードの 40-70%）
- マスク外のセルは `TILE_VOID`（壁でも通路でもない虚無。真っ暗に表示）
- **島（連結成分）**: 同じ階に複数の独立した島が存在しうる
- **廊下骨格**: 島のDFSノードの20-30%をランダムウォークで選び、2セル幅の廊下を先に生成
- **部屋配置**: 2x2〜5x4 の部屋を廊下隣接優先で配置（最大120回試行、重複禁止、壁1枚隣接は許可）
- 各島で独立に DFS 迷路生成。部屋内・廊下ノードは pre-carved として扱い、残りの領域をDFSで細い通路として接続
- 同一階の島同士は直接行き来不可（迷子感の演出）
- 階段で別の階の別の島に出ることで、島を渡り歩く構造
- サイズはガラガラの回転回数で決定（15x13 〜 121x91）
- ゴールは最大の島の最も遠い空きマスに自動配置
- ビスケットは空きマスの約 30% にランダム配置
- 内壁の約 15% がランダムに窓/障子に置換される（窓 70%、障子 30%）

## Visual Design

### Rendering Pipeline

1. フレームバッファをクリア
2. レイキャスティング（列ごとに 1 本の ray）
3. 床・天井の描画（パースペクティブ正確な座標計算）
4. 壁の描画（テクスチャ付き）
5. スプライトの描画（AA パターン、深度テスト付き）
6. ミニマップのオーバーレイ（半透明ブレンド）
7. HUD（空腹バー、メッセージ）
8. デルタフラッシュでターミナルに出力

### Wall Textures (Procedural)

- 板壁の縦溝（タイル端で暗くなる）
- 長押（なげし）— 壁の上部 35% に水平レール
- 幅木（はばき）— 壁の下部に暗い帯
- 木目模様 — 壁面水平位置に基づく縦の波紋
- 横板の継ぎ目 — 壁を 3 分割する薄い線
- タイル座標ハッシュで壁ごとに微妙な色違い

### Floor & Ceiling (Mode7-Style)

- パースペクティブ正確なワールド座標計算
- 床: 畳/板張りのグリッド線 + 板目ライン + タイルごとの色ムラ
- 天井: 控えめなパネルグリッド
- 距離に応じてパターン密度が自然に変化（近い = 粗い、遠い = 密）
